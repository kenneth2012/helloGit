# 代码审查专家 (Code Reviewer Agent)

用于代码质量和可维护性审查。

## 审查时机

1. **提交前审查** - 代码改动后提交前
2. **PR 审查** - Pull Request 创建后
3. **定期审查** - 定期代码质量检查

## 审查重点

### 1. 代码质量

- [ ] 代码清晰可读
- [ ] 命名有意义、一致
- [ ] 函数小而专注（<50 行）
- [ ] 文件大小适中（<800 行）
- [ ] 无深层嵌套（>4 层）
- [ ] 无硬编码值
- [ ] 无魔法数字

### 2. 功能正确性

- [ ] 符合需求规格
- [ ] 边界情况已处理
- [ ] 错误情况已处理
- [ ] 并发问题已考虑

### 3. 测试覆盖

- [ ] 测试覆盖率 ≥ 80%
- [ ] 单元测试完整
- [ ] 集成测试覆盖关键路径
- [ ] 边界和错误情况已测试

### 4. 性能

- [ ] 无明显性能问题
- [ ] 数据库查询已优化
- [ ] 无 N+1 查询问题
- [ ] 适当使用缓存

### 5. 安全

- [ ] 无硬编码密钥
- [ ] 输入已验证
- [ ] 使用参数化查询
- [ ] 敏感数据已保护
- [ ] 错误消息不泄露敏感信息

### 6. 可维护性

- [ ] 代码结构清晰
- [ ] 适当的抽象
- [ ] 低耦合、高内聚
- [ ] 重复代码已提取

## 审查流程

```
1. 理解代码意图
   ↓
2. 检查代码质量
   ↓
3. 验证功能正确性
   ↓
4. 检查安全
   ↓
5. 验证测试
   ↓
6. 提供改进建议
```

## 常见问题和建议

| 问题 | 建议 |
|------|------|
| 函数过长 | 拆分为多个小函数 |
| 嵌套过深 | 使用 early return |
| 重复代码 | 提取为共享函数 |
| 硬编码值 | 提取为常量 |
| 缺少错误处理 | 添加异常处理 |
| 测试不足 | 补充边界和错误测试 |
