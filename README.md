# AI 编程助手技能库学习笔记

本目录包含三个主流 AI 编程助手技能库的学习笔记，以及它们在 Trae Solo、CodeBuddy 等平台上的适配指南。

## 📚 文件列表

1. **[01-karpathy-skills.md](./01-karpathy-skills.md)** - Andrej Karpathy Skills
   - 作者：Forrest Chang
   - 核心理念：先想再写、简单优先、精准改动、目标驱动
   - 适合：日常编码，减少 AI 常见错误

2. **[02-gstack.md](./02-gstack.md)** - gstack
   - 作者：Garry Tan (YC CEO)
   - 核心理念：不同任务用不同认知模式，角色化思考
   - 适合：产品驱动的开发，从 0 到 1 项目

3. **[03-superpowers.md](./03-superpowers.md)** - Superpowers
   - 作者：Jesse Vincent (@obra)
   - 核心理念：Process over Prompt（流程大于提示词）
   - 适合：需要严格工程规范的项目，团队协作

## 🆚 三种方法论对比

| 方法论 | 核心理念 | 适用场景 | Star 数 |
|--------|----------|----------|---------|
| **Karpathy Skills** | 先想再写，简单优先，精准改动，目标驱动 | 日常编码，减少 AI 常见错误 | 105,000+ |
| **gstack** | 不同任务用不同认知模式，角色化思考 | 产品驱动的开发，从 0 到 1 项目 | 100,000+ |
| **Superpowers** | Process over Prompt，强制 TDD 和流程纪律 | 需要严格工程规范的项目，团队协作 | 124,000+ |

## 🛠️ 平台适配

所有三个技能库都提供了在以下平台的适配指南：

- **Trae Solo** - 通过 Rules 功能配置
- **Claude Code** - 官方插件支持
- **Cursor** - 规则文件支持
- **CodeBuddy** - 通过系统提示词配置

## 📖 快速开始

根据你的需求选择合适的方法论：

1. **如果是日常编码，想减少 AI 犯错** → 从 Karpathy Skills 开始
2. **如果是产品驱动的项目，从 0 到 1** → 试试 gstack 的角色化思考
3. **如果需要严格的工程纪律、团队协作** → Superpowers 的 TDD 流程可能更适合

你也可以根据项目的不同阶段组合使用它们！
