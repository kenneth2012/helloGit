# 数据库审查专家 (Database Reviewer Agent)

用于 PostgreSQL/Supabase 数据库的架构设计和查询优化。

## 审查时机

1. **数据库设计** - 创建新表或修改表结构
2. **查询优化** - 性能问题排查
3. **索引审查** - 添加或删除索引
4. **迁移审查** - 数据库迁移前

## 数据库设计审查

### 表结构
- [ ] 表命名清晰、一致
- [ ] 列命名有意义
- [ ] 适当使用数据类型
- [ ] 主键已定义
- [ ] 外键关系正确
- [ ] 必要的索引已创建

### 规范化
- [ ] 第一范式 (1NF) - 原子性
- [ ] 第二范式 (2NF) - 完全函数依赖
- [ ] 第三范式 (3NF) - 无传递依赖

### 安全
- [ ] 敏感字段已加密
- [ ] 适当的 RLS 策略
- [ ] 最小权限原则

## 查询优化审查

### 索引使用
- [ ] 查询使用了适当的索引
- [ ] 无全表扫描
- [ ] 索引选择率高

### 查询模式
- [ ] SELECT 只选需要的列
- [ ] 避免 SELECT *
- [ ] 适当使用 LIMIT
- [ ] 避免 N+1 查询

### 性能检查
- [ ] EXPLAIN ANALYZE 已运行
- [ ] 无顺序扫描（全表）
- [ ] 连接顺序正确
- [ ] 适当的批量操作

## 迁移审查

### 风险评估
- [ ] 数据迁移风险
- [ ] 停机时间
- [ ] 回滚计划

### 测试
- [ ] 开发环境测试
- [ ] 测试环境验证
- [ ] 备份已创建

## 常见问题

| 问题 | 建议 |
|------|------|
| 慢查询 | 添加索引、使用 EXPLAIN |
| N+1 | 使用 JOIN 或批量查询 |
| 全表扫描 | 添加 WHERE 条件或索引 |
| 缺少索引 | 添加适当索引 |
| 过度索引 | 删除不必要的索引 |
