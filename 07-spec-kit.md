# GitHub Spec Kit 学习笔记

## 项目基本信息

- **仓库地址**: https://github.com/github/spec-kit
- **作者**: GitHub 官方
- **开源协议**: MIT License
- **核心定位**: 规范驱动的 AI 开发框架，通过 4 阶段流程（Specify → Plan → Tasks → Implement）实现高质量软件开发
- **官方文档**: https://github.github.io/spec-kit/

## 项目简介

GitHub Spec Kit 是一个开源工具包，旨在让组织专注于产品场景而不是编写无差异化的代码。通过 Spec-Driven Development (SDD) 规范驱动开发，帮助团队在 AI 辅助编程中保持上下文一致性和高质量输出。

**核心理念**: 规格说明书成为可执行的，直接生成可工作的实现，而不仅仅是指导代码编写。

## 核心哲学

| 原则 | 说明 |
|------|------|
| **意图驱动开发** | 规格说明书先定义"做什么"，再讨论"怎么做" |
| **丰富的规格创建** | 使用护栏和组织原则来创建规范 |
| **多步细化** | 通过多步骤而非一次性提示生成代码 |
| **重依赖 AI 能力** | 利用先进 AI 模型进行规格解读 |

## 4 阶段开发流程

### Phase 1: Specify（规格说明）

**目标**: 将想法转化为结构化的规格说明书

```
/speckit.specify 构建一个帮助我整理照片的应用。相册按日期分组，可以在主页通过拖放重新组织...
```

**关键活动**:
- 明确用户需求和期望结果
- 定义功能边界和约束
- 建立验收标准
- 识别关键用户场景

### Phase 2: Plan（技术规划）

**目标**: 将规格转化为可执行的技术方案

```
/speckit.plan 应用使用 Vite，最小化依赖库。尽可能使用原生 HTML、CSS 和 JavaScript...
```

**关键活动**:
- 选择技术栈
- 定义架构模式
- 设计数据流
- 规划部署策略

### Phase 3: Tasks（任务分解）

**目标**: 将规划分解为可执行的小任务

```
/speckit.tasks 验证任务 -> spec-verify-tasks
```

**关键活动**:
- 分解为 2-5 分钟可完成的小任务
- 定义每个任务的验证标准
- 识别任务依赖关系
- 制定执行顺序

### Phase 4: Implement（实现）

**目标**: 按照任务列表逐步实现

**关键活动**:
- 按顺序执行任务
- 每个任务后进行验证
- 保持与规格的一致性
- 迭代优化

## 支持的项目类型

### Greenfield Projects（绿地项目）

从零开始的新项目，适合创新性探索。

### Brownfield Projects（棕地项目）

在现有代码库上添加功能，尊重既有架构。

### Exploratory Projects（探索性项目）

从同一规格生成多个实现方案，比较不同优化目标。

### Refactoring Projects（重构项目）

将重构目标作为规格文档，生成渐进式重构计划。

## 核心命令

### /speckit.specify

描述要构建的功能，聚焦"做什么"和"为什么"。

```
/speckit.specify 构建一个任务管理应用，允许用户创建项目、添加成员、分配任务、评论和移动任务...
```

### /speckit.plan

提供技术栈和架构选择。

```
/speckit.plan 应用使用 .NET Aspire + Postgres。前端使用 Blazor Server 实现拖放功能...
```

### /speckit.tasks

创建可执行的任务列表。

```
/speckit.tasks 验证任务 -> spec-verify-tasks
```

### /speckit.verify-tasks

验证任务完成情况。

```
/speckit.verify-tasks 读取 SPEC.md 和 plan 目录中的任务，确保所有任务都已完成...
```

## 适用场景

### 适用场景

1. **AI 辅助开发**
   - 需要在多轮对话中保持上下文
   - 想提高 AI 生成代码的质量和一致性

2. **复杂项目**
   - 多功能系统需要架构规划
   - 需要明确的功能边界和验收标准

3. **团队协作**
   - 需要统一开发流程
   - 需要可追溯的决策记录

4. **重构项目**
   - 需要渐进式重构
   - 需要保持现有功能稳定

### 不适用场景

1. **简单任务**
   - 单个文件修改
   - 快速脚本编写
   - 拼写错误修正

2. **探索阶段**
   - 概念验证
   - 技术调研
   - 原型开发

3. **紧急修复**
   - 生产环境紧急 bug
   - 时间敏感的小改动

## 快速开始模板

### 新项目

```markdown
# SPEC.md

## 项目：Taskify 任务管理平台

## 概述
团队生产力平台，支持看板式任务管理。

## 用户故事
1. 用户可以选择预定义账户登录
2. 用户可以创建和管理项目
3. 用户可以在看板上拖放任务卡片
4. 用户可以评论和分配任务

## 功能列表
- [ ] 用户选择登录
- [ ] 项目管理（创建、查看）
- [ ] 看板视图（To Do、In Progress、In Review、Done）
- [ ] 任务拖放
- [ ] 任务评论
- [ ] 任务分配

## 技术约束
- 纯前端实现，无需后端
- 本地存储数据
- 支持拖放的 UI 库

## 验收标准
- [ ] 所有功能在浏览器中正常工作
- [ ] 任务可以在列间拖放
- [ ] 评论功能正常
- [ ] 数据持久化到本地存储
```

## 为什么选择 Spec Kit

1. **AI 原生设计**: Spec Kit 本身就是为 AI 辅助开发而生
2. **结构化流程**: 4 阶段流程确保开发有章可循
3. **持久化上下文**: 规格说明书在每次对话中保持一致性
4. **GitHub 官方**: 背靠 GitHub 的工程最佳实践
5. **可扩展性**: 支持社区扩展和预设模板
