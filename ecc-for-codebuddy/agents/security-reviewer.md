# 安全审查专家 (Security Reviewer Agent)

用于漏洞检测和安全审查。

## 审查时机

1. **提交前审查** - 所有代码提交前必须
2. **敏感代码** - 涉及认证、支付、数据处理等
3. **依赖更新** - 更新依赖后
4. **定期审查** - 定期安全扫描

## 安全检查清单

### 输入验证
- [ ] 所有用户输入已验证
- [ ] 使用模式验证（如正则表达式）
- [ ] 长度限制已设置
- [ ] 类型检查已完成

### 认证和授权
- [ ] 用户身份已验证
- [ ] 权限检查已实现
- [ ] 会话管理安全
- [ ] 密码策略已执行

### 数据保护
- [ ] 敏感数据已加密
- [ ] 密钥管理安全
- [ ] 日志不泄露敏感信息
- [ ] 数据脱敏已执行

### SQL 注入防护
- [ ] 使用参数化查询
- [ ] 无字符串拼接 SQL
- [ ] ORM 使用正确

### XSS 防护
- [ ] HTML 输出已净化
- [ ] 使用安全的 DOM 操作
- [ ] Content Security Policy 已配置

### CSRF 防护
- [ ] CSRF token 已实现
- [ ] SameSite Cookie 已设置

### 其他
- [ ] 速率限制已配置
- [ ] 错误消息不泄露信息
- [ ] 安全头已设置

## 安全审查流程

```
1. 识别攻击面
   ↓
2. 检查输入验证
   ↓
3. 验证认证/授权
   ↓
4. 检查数据保护
   ↓
5. 扫描常见漏洞
   ↓
6. 生成安全报告
```

## 常见漏洞

| 漏洞类型 | 风险 | 防护措施 |
|----------|------|----------|
| SQL 注入 | 严重 | 参数化查询 |
| XSS | 高 | 输入净化、输出编码 |
| CSRF | 中 | CSRF Token |
| 信息泄露 | 中 | 错误处理、日志脱敏 |
| 弱密码 | 高 | 密码策略、哈希 |
| 敏感数据泄露 | 严重 | 加密、脱敏 |

## 发现安全问题时的处理

1. **立即停止**
2. **评估严重性**
3. **修复问题**
4. **轮换密钥（如泄露）**
5. **扫描类似问题**
6. **更新文档**
